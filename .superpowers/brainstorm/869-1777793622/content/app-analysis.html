Kritische Lücken 🔴

💾
Dateien in localStorage — Zeitbombe
Base64-kodierte Dateien im localStorage. Limit ~5–10 MB. Schon 2–3 PDFs = App crasht. Tauri Filesystem API oder IndexedDB nötig.
📊
Dashboard = nur Skeleton
DashboardPane zeigt nur Shimmer-Placeholder. Keine echten KPI-Karten, keine Charts, keine Aktivitäten.
↩️
Kein Undo/Redo
Gelöschte Notizen, Todos, KPIs sind weg. Nur Bestätigungs-Modals als Schutz.
📝
Markdown-Parser zu simpel
Kein Syntax-Highlighting, keine Tabellen, keine verschachtelten Listen. remark-gfm ist installiert aber ungenutzt.

Store Performance — Status

Der Zustand liegt in einem einzigen Zustand-Objekt mit localStorage-Persistenz. Bei jedem State-Update wird das gesamte Objekt serialisiert & gespeichert — inkl. aller Base64-Dateien. Selektoren (useStore(s => s.field)) sind gut eingesetzt, aber die Persistenz-Logik ist der Flaschenhals.

Außerdem: keine Pagination für lange Listen, kein Memoization für gefilterte/sortierte Listen.
